Executive Summary

TryAndFit is a web-based AI virtual try-on platform that lets shoppers see how any outfit looks on their own body before purchasing — using just a selfie or full-body photo. Users can submit an outfit image directly, or paste a product link from Indian and global e-commerce platforms (Myntra, Flipkart, Amazon, Meesho, AJIO, Instagram, or any website), and the AI generates a photorealistic composite showing them wearing the outfit.

🎯

Core Value Prop

Reduce fashion purchase anxiety and returns by letting users virtually wear any garment before buying.

🌏

Target Market

India-first (Myntra, Flipkart, Meesho, Amazon.in, AJIO) with global web support via URL scraping.

🤖

AI Engine

Fashn.ai virtual try-on API — purpose-built for garment transfer on real human photos.

⚡

Infrastructure

Next.js on Vercel, Supabase (Postgres + Edge Functions + Storage), GCP bucket for image assets.

Problem & Opportunity

The Problem

Online fashion shoppers in India face a fundamental trust gap: product photos show models — not them. This leads to high return rates (estimated 25–40% for fashion in India), purchase hesitation, and wasted money. No major Indian platform offers AI try-on natively.

The Opportunity

📦

$10B+

Indian online fashion market (2024), growing 15% YoY

↩️

30%+ Returns

Average fashion return rate driven by "doesn't look right on me"

🛍️

No Native Try-On

Myntra, Meesho, Flipkart fashion — none offer real AI try-on today

User Pain Points

  • Can't tell how a garment looks on their body type or skin tone
  • Model photos don't represent the average Indian body
  • Ordering multiple sizes to try = expensive and wasteful
  • Instagram outfit inspiration can't be "tried on" instantly

Technical Architecture

React 18, TailwindCSS, ShadCN UI. Deployed to Vercel Edge Network. ISR for landing pages. CSR for the app UI.

Serverless functions for: URL scraping, image pre-processing, Fashn.ai API calls, result storage. Runs at edge for low latency.

Users, try-on jobs, results, wardrobe, credits, feedback. Row Level Security (RLS) enforced on all tables.

Supabase Storage for user uploads (person photos, outfit photos). GCP Cloud Storage bucket for output result images (CDN-served). Signed URLs for private access.

Dedicated virtual garment try-on model. Accepts person image + garment image → outputs photorealistic composite. Best-in-class for fashion specifically.
🔐
Auth — Supabase Auth
Google OAuth + Magic Link email. JWT tokens. Guest mode supported for first try-on without signup.
📊
Analytics — Vercel Analytics + Supabase logs
Page views, try-on funnel, conversion rate, API latency.

AI & ML Pipeline

6.1 Why Fashn.ai

Fashn.ai is a dedicated AI virtual try-on API purpose-built for fashion — not a general image generation model. It understands garment structure, fabric drape, body pose, and lighting context, producing far superior results compared to generic diffusion models for this specific use case.

6.2 Input Requirements

InputRequiredFormatNotes
Person photoYesJPG/PNG, ≥512pxFull body preferred, clear background helps
Garment imageYesJPG/PNG, ≥512pxFlat lay or on-model both work
Garment categoryOptionaltop / bottom / dressAuto-detected if not provided

6.3 Pre-Processing (Edge Function)

  • Resize and normalize images to 768×1024 for optimal API performance
  • Strip EXIF metadata for privacy
  • Convert to JPEG (90% quality) to reduce payload size
  • Validate: faces present in person photo, garment visible in outfit photo

6.4 Async Job Queue

Try-on requests are processed asynchronously. The flow:

  1. Frontend submits job → gets job_id immediately
  2. Edge Function queues Fashn.ai API call
  3. Frontend polls /api/jobs/[job_id] every 2 seconds
  4. On completion, result URL is returned and stored
  5. Realtime update via Supabase Realtime (optional enhancement)

6.5 Fallback Strategy

⚠️ If Fashn.ai returns an error (bad image quality, NSFW flagged, timeout): display a helpful error message with retry option. Log failures to Supabase for quality monitoring. Do not charge credits on failed jobs.

Security & Privacy

  • Row Level Security (RLS) — All Supabase tables have RLS policies; users can only access their own data
  • Photo privacy — Person photos are stored with private signed URLs (1hr expiry); never publicly accessible
  • EXIF stripping — GPS and device metadata stripped from all uploaded images before storage
  • Auto-deletion — Person photos are deleted after 24 hours (configurable). Results retained for 90 days
  • No face recognition — Person photos are passed to Fashn.ai only for garment overlay; no facial data is stored or analyzed
  • HTTPS only — All traffic over TLS 1.3; HSTS headers set
  • GDPR-ready — Data export and deletion endpoints for user account management
  • Content moderation — Fashn.ai has built-in NSFW detection; we additionally validate uploads for appropriate content
  • API key security — Fashn.ai key stored as Supabase/Vercel secret; never exposed to frontend
Important: Clearly communicate to users in the UI and Terms of Service how their photos are used, stored, and deleted. This is critical for trust in a photo-sensitive app.

Risks & Mitigations

RiskLikelihoodImpactMitigation
Myntra/Flipkart blocks scraping (anti-bot) Medium High Use rotating proxies; partner with platforms for API access; fall back to user uploading product image manually
Fashn.ai quality issues on Indian skin tones / body types Medium Medium Run quality benchmarks pre-launch; collect feedback; explore fine-tuning or alternative models if needed
Fashn.ai API downtime Low High Queue jobs with retry logic; show "processing" state gracefully; evaluate Replicate as backup
High GCP egress costs at scale Medium Medium Use CDN caching aggressively; compress images; move to Cloudflare R2 for cost savings at scale
Privacy concerns around user photos Medium High Auto-delete photos within 24h; clear privacy policy; no facial data storage; signed private URLs only
Misuse (uploading photos of others) Medium Medium Terms of Service, NSFW filtering, reporting mechanism, rate limiting, account bans
